Notable changes to Schemic, newest first. Each entry is written from your side — what changes for code that uses the library.

<Callout variant="caution" title="Alpha: expect breaking changes">
Schemic is `0.1.0-alpha`. The API still changes between releases. Pin an exact version and read this page before upgrading. Breaking changes are marked **Breaking**.
</Callout>

The packages release in lockstep — one version across `@schemic/core`, `@schemic/cli`, `@schemic/surrealdb`, and the rest. This page covers the SurrealDB driver and the shared core/CLI; the full per-version history for every package lives in the repo [CHANGELOG.md](https://github.com/schemichq/schemic/blob/main/CHANGELOG.md).

## 0.1.0-alpha.22

### Breaking

- **`defineAccess` requires an explicit scope.** `defineAccess(name)` now returns a builder exposing only `.onDatabase()` / `.onNamespace()` — you pick one before the kind (`.record()` / `.jwt()` / `.bearer()`), enforced at compile time. The silent `ON DATABASE` default is gone: access scope is a security boundary, so it must be chosen.
- **`.reference()` is renamed `.$reference()`** on field builders, matching the other `$`-prefixed DDL clauses (`.$unique()`, `.$index()`, `.$default()`).
- **`.$fulltext({ bm25 })` accepts only `[k1, b]`.** `bm25: true` is dropped — it was a no-op, since BM25 is always on.

### Added

- **Full-text and vector indexes** are now expressible on a field: `.$fulltext()`, `.$hnsw()`, `.$diskann()` — plus record-reference links via `.$reference(...)` with `ON DELETE` behaviour.
- **`DEFINE ACCESS` authoring** (non-secret): `.comment()` and `.withRefresh()`, with a guard that keeps `RECORD` access database-only.
- **`DEFINE EVENT` gains `async` and `comment`** on `.event()` / `defineEvent`.
- **`defineAnalyzer().function(cb, name?)`** takes an optional custom name for the auto-defined function.
- **Author-time field validation:** `schemic gen` rejects invalid `DEFINE FIELD` clause combinations with a clear error instead of failing at apply.

### Fixed

- `defineAnalyzer().function()` now emits its auto-defined `DEFINE FUNCTION` (previously the analyzer referenced a missing one).
- `schemic gen` shows the rendered migration **before** the name prompt, so you review the DDL while naming it.

## 0.1.0-alpha.21

### Breaking

- **Package split (M0.3).** The authoring import `@schemic/surrealdb` is now side-effect-free — `s.*`, `define*`, and `surql` only. The connection factory moved to **`@schemic/surrealdb/connection`** (update the `surrealConnection` import in your `schemic.config.ts`), and the driver implementation to `@schemic/surrealdb/driver` (engine/CLI only). The query builder stays at `@schemic/surrealdb/query`.

### Added

- **`defineFunction(args).returns(R).call(db, args)`** — call a defined database function as typed code: arguments encoded via the param schemas, the result decoded through `.returns(R)`.

## 0.1.0-alpha.20

### Added

- **`schemic pull --watch`** polls the live database (`--interval`, default 2s) and re-pulls as it changes.
- **`callFunction`** in `@schemic/core/query` invokes a defined database function and decodes the result through `.returns(R)`.

### Fixed

- `defineAnalyzer().filters()` no longer dedupes — duplicate filters pass through verbatim.

## 0.1.0-alpha.19

### Fixed

- `defineAnalyzer().tokenizers()` no longer dedupes — duplicate tokenizers pass through verbatim.

## 0.1.0-alpha.18

### Breaking

- **`.flexible()` / `.loose()` / `.strict()` are object-only** — a compile error on non-object fields (previously a silent no-op).
- **`defineAnalyzer`'s config-object form is dropped** in favour of the fluent builder (`.tokenizers(...).filters(...)`).

### Added

- **Typed `select()` query builder** at `@schemic/surrealdb/query` — `where` / `orderBy` / `limit` / `.return` projection, decode-by-default, `.raw()` to opt out.
- **Full `DEFINE ANALYZER` coverage** with a fluent `defineAnalyzer` builder (tokenizers, filters, function, comment).
- **`@schemic/core/query`** — the neutral query toolkit the driver builders compose on.
