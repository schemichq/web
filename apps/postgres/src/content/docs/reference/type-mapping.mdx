This is the exhaustive matrix from each `s` builder to the Postgres column it emits, with a support status. The `s` vocabulary is Postgres-native: alongside the Zod drop-ins (`s.string`, `s.number`) there are precise pg types (`text`, `varchar`, `numeric`, `timestamptz`, …). For how types compose into DDL see [from schema to DDL](/docs/concepts/schema-to-ddl).

## Not yet covered

These Postgres features are not expressible in the schema today. Do not rely on them yet:

- **Native `ENUM` / `DOMAIN`** (`CREATE TYPE`) — `s.enum` projects to `text` for now.
- **Extension-backed types** — citext, PostGIS, pgvector, and other types that require an extension.
- **Views and materialized views, functions, procedures, triggers, standalone sequences.**
- **Multiple schemas** — everything lives in `public`.
- **RLS policies, roles, and grants.**
- **Composite / multi-column foreign keys, and keys to a non-`id` column.**
- **Non-btree index methods** (gin, gist, brin, hash) and **partial / expression indexes**; **`EXCLUDE` constraints.**
